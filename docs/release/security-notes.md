# Security Notes

This document tracks accepted security exceptions and license-inventory caveats
for the sophia-cli release line. Every entry is human-reviewed; renew the
review on every minor release.

## Severity gates (D-M9-11)

- **govulncheck**: `make vuln` blocks on **reachable HIGH/CRITICAL** vulnerabilities
  in code paths the binary actually calls. Non-reachable advisories surface as
  warnings only and may be logged in the "govulncheck non-reachable" section
  below if relevant.
- **gosec**: `make security` blocks on **HIGH severity** findings. MEDIUM and
  LOW findings require per-finding justification below â€” no blanket waivers.
  False positives at the call site use `// #nosec G123 -- reason` annotations
  with the reason inline.

## License inventory caveats (D-M9-15)

`THIRD_PARTY_LICENSES.md` is generated by `make licenses`, which prefers
`go-licenses report` and falls back to `scripts/licenses-fallback.sh` (a
`go list -m -json all` inventory) when `go-licenses` cannot run on the current
toolchain.

**Status at v0.1.0:** the fallback path is active. `go-licenses` (v1.x) does
not currently support the Go 1.26.x toolchain stack the project targets via
`go.mod`'s `toolchain go1.26.2` directive. The fallback inventory enumerates
every transitive module by path + version but DOES NOT resolve license names â€”
each module is marked `(verify manually)`.

**Acceptance for v0.1.0:**
- The full module set is enumerated; nothing is hidden.
- Every module is sourced from public repositories under widely-recognized
  licenses (Apache-2.0, MIT, BSD-3-Clause, MPL-2.0). The team's manual review
  of the inventory at v0.1.0 found no GPL-family contaminations.
- M10 will revisit once `go-licenses` supports the Go 1.26.x toolchain (or
  earlier if a compatible alternative emerges) and replace the inventory with
  fully-resolved license fields.
